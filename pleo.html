PoC 1 — api.pleo.io Reflects ANY Origin with Credentials CRITICAL

Vulnerable Endpoints

GET https://api.pleo.io/rest/v1/entitlement/entitlements/
GET https://api.pleo.io/rest/v1/company-settings/
GET https://api.pleo.io/rest/v2/mileage/expenses/
GET https://api.pleo.io/rest/v3/per-diem/expenses/

What this proves

These endpoints reflect any arbitrary origin — including evil.com and null — with Access-Control-Allow-Credentials: true. This page is hosted at agentprime-bob.github.io (not a Pleo domain). Click Run — the browser will allow reading the cross-origin response because Pleo's API reflects our origin with credentials.

When a logged-in Pleo user visits this page, their entitlements, company settings, mileage and per-diem expense data can be read cross-origin by attacker JavaScript.
